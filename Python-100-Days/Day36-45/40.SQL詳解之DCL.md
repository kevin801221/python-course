## SQL詳解之DCL

資料庫伺服器通常包含了非常重要的資料，可以透過訪問控制來確保這些資料的安全，而 DCL 就是解決這一問題的，它可以為指定的使用者授予訪問許可權或者從指定使用者處召回指定的許可權。DCL 對資料庫管理員來說非常重要，因為使用者許可權的管理關係到資料庫的安全。簡單的說，我們可以透過 DCL 允許受信任的使用者訪問資料庫，阻止不受信任的使用者訪問資料庫，同時還可以透過 DCL 將每個訪問者的的許可權最小化（讓訪問者的許可權剛剛夠用）。

### 建立使用者

我們可以使用下面的 SQL 來建立一個使用者併為其指定訪問口令。

```SQL
CREATE USER 'wangdachui'@'%' IDENTIFIED BY 'Wang.618';
```

上面的 SQL 建立了名為 wangdachui 的使用者，它的訪問口令是 Wang.618，該使用者可以從任意主機訪問資料庫伺服器，因為 @ 後面使用了可以表示任意多個字元的萬用字元 %。如果要限制 wangdachui 這個使用者只能從 192.168.0.x 這個網段的主機訪問資料庫伺服器，可以按照下面的方式來修改 SQL 語句。

```SQL
DROP USER IF EXISTS 'wangdachui'@'%';

CREATE USER 'wangdachui'@'192.168.0.%' IDENTIFIED BY 'Wang.618';
```

此時，如果我們使用 wangdachui 這個賬號訪問資料庫伺服器，我們幾乎不能做任何操作，因為該賬號沒有任何操作許可權。

### 授予許可權

我們用下面的語句為 wangdachui 授予查詢 school 資料庫學院表（`tb_college`）的許可權。

```SQL
GRANT SELECT ON `school`.`tb_college` TO 'wangdachui'@'192.168.0.%';
```

我們也可以讓 wangdachui 對 school 資料庫的所有物件都具有查詢許可權，程式碼如下所示。

```SQL
GRANT SELECT ON `school`.* TO 'wangdachui'@'192.168.0.%';
```

如果我們希望 wangdachui 還有 insert、delete 和 update 許可權，可以使用下面的方式進行操作。

```SQL
GRANT INSERT, DELETE, UPDATE ON `school`.* TO 'wangdachui'@'192.168.0.%';
```

如果我們還想授予 wangdachui 執行 DDL 的許可權，可以使用如下所示的 SQL。

```SQL
GRANT CREATE, DROP, ALTER ON `school`.* TO 'wangdachui'@'192.168.0.%';
```

如果我們希望 wangdachui 賬號對所有資料庫的所有物件都具備所有的操作許可權，可以執行如下所示的操作，但是一般情況下，我們不會這樣做，因為我們之前說過，許可權剛剛夠用就行，一個普通的賬號不應該擁有這麼大的許可權。

```SQL
GRANT ALL PRIVILEGES ON *.* TO 'wangdachui'@'192.168.0.%';
```

### 召回許可權

如果要召回 wangdachui 對 school 資料庫的 insert、delete 和 update 許可權，可以使用下面的操作。

```SQL
REVOKE INSERT, DELETE, UPDATE ON `school`.* FROM 'wangdachui'@'192.168.0.%';
```

如果要召回所有的許可權，可以按照如下所示的方式進行操作。

```SQL
REVOKE ALL PRIVILEGES ON *.* FROM 'wangdachui'@'192.168.0.%';
```

需要說明的是，由於資料庫可能會快取使用者的許可權，可以在授予或召回許可權後執行下面的語句使新的許可權即時生效。

```SQL
FLUSH PRIVILEGES;
```
